# Chinese Webnovel Writing

## Overview

Use this skill for Chinese web novel creation and revision. It focuses on practical output: premise shaping, chapter planning, opening hooks, character power systems, pacing, cliffhangers, and serial consistency.
All generated content from this skill must be in Chinese unless the user explicitly asks for another language.

## Core Workflow

1. Identify the target genre and monetization style.
   - Examples: fantasy, xianxia, urban superpower, military, mystery, romance, game-lit, infinite flow.
   - Determine whether the story should optimize for fast hooks, steady suspense, emotional payoff, or combat escalation.

2. Define the core reading engine.
   - State the protagonist's main goal, recurring conflict, and primary reward loop.
   - Make the story's "why keep reading" explicit.
   - Typical engines: revenge, rise-from-zero, hidden identity, system growth, sect competition, family drama, survival.

3. Build a serial-safe premise.
   - One-sentence premise.
   - Protagonist drawback.
   - Opposing force.
   - Escalation path for at least 30-100 chapters.
   - A clear early turning point in the first 3 chapters.

4. Design the opening.
   - Start with conflict, not background.
   - Give the protagonist a visible problem, a concrete desire, and an immediate obstacle.
   - End the opening scene with tension, choice, reveal, or reversal.

5. Plan chapter rhythm.
   - Each chapter should usually contain: goal, friction, reaction, and hook.
   - Mix information, action, and payoff.
   - Avoid long exposition unless it directly raises stakes.

6. Check serial retention.
   - Every arc should answer one question and create a bigger one.
   - Track progression, reveals, enemies, relationships, and side goals.
   - Make sure the story can keep producing chapter-ending hooks.

7. Prevent drift.
   - Keep a fixed story bible, chapter tracker, and plot-future log.
   - Any new scene must serve the current volume goal, the protagonist arc, or the long-term payoff.
   - If a scene only adds noise, delay, or vibes without movement, cut it.

## Output Patterns

When the user asks for a novel, produce one of these formats:

- `Idea pitch`: genre, premise, target audience, and selling points.
- `Outline`: world setting, protagonist, core conflict, power system, major arcs, and chapter beats.
- `Chapter draft`: scene-by-scene prose with strong hooks and readable cadence.
- `Revision pass`: diagnose pacing, dull scenes, weak conflict, unclear stakes, and repetitive wording.
- `Revise mode`: fix drift, flattening, weak scenes, awkward dialogue, and AI-like prose in an existing draft.
- `Longform build`: design a multi-volume structure, role web, escalation map, and serial pacing model for a 300-500 chapter novel.
- `Continuity control`: prevent drift, keep chapter-level alignment, and manage foreshadowing/reward timing across a long run.

## Practical Rules

- Prefer concrete nouns and active verbs.
- Keep the protagonist's goal visible in every major scene.
- Make conflicts specific, not abstract.
- Use reversals, reveals, and consequences to end chapters.
- If the user asks for "大神风格," imitate structure and rhythm, not a living author's exact prose.
- If the user asks for a long-form novel, build around volume-level turns, chapter-level hooks, and character-caused reversals rather than a single master template.
- If the user asks for a long novel, first lock the premise, then the arc plan, then chapter-by-chapter execution.
- Before drafting a new chapter batch, confirm the target volume goal, current state of each major relationship, and the next payoff due.

## Useful Templates

### One-Sentence Premise

`When [inciting event], [protagonist] must [goal] or [cost], while [opposing force] escalates the pressure.`

### Chapter Beat

`Goal -> Obstacle -> Response -> Twist/Hook`

### Arc Beat

`Setup -> First win -> Bigger threat -> Mid-arc reversal -> Low point -> Final payoff`

## Revision Checklist

- Is the opening immediate?
- Is the protagonist's desire obvious?
- Does each chapter end with tension?
- Is there progression in status, ability, relationship, or mystery?
- Are the scenes doing one job each?
- Does the draft still match the longform framework and the style banlist?
